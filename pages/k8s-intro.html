Kubernetes Introduction

Master container orchestration and deployment at scale

What is Kubernetes?

Kubernetes (K8s) is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications. It was originally designed by Google and is now maintained by the Cloud Native Computing Foundation.

Why Use Kubernetes?

  • Automated Deployment: Deploy containers across a cluster automatically
  • Self-Healing: Automatically restart failed containers and replace unhealthy nodes
  • Load Balancing: Distribute network traffic across containers
  • Storage Orchestration: Automatically mount and manage storage systems
  • Horizontal Scaling: Scale your applications up or down with a single command
  • Rolling Updates: Update applications without downtime

Kubernetes Architecture

Master Node (Control Plane): Manages the Kubernetes cluster, scheduling, and API server.

Worker Nodes: Run the actual containerized applications (pods).

Pods: The smallest deployable unit in Kubernetes, typically containing one or more containers.

Basic Kubernetes Objects

Deployment: Describes a desired state for pods and replica sets. The controller then changes the actual state to the desired state at a controlled rate.

Service: An abstract way to expose applications running on pods as a network service.

ConfigMap: Stores non-confidential configuration data in key-value pairs.

Secret: Securely stores sensitive data like passwords and API keys.

Essential kubectl Commands

# Get cluster information
kubectl cluster-info

# Create resources from YAML files
kubectl apply -f deployment.yaml

# List pods
kubectl get pods

# Describe a pod
kubectl describe pod <pod-name>

# View logs
kubectl logs <pod-name>

# Execute commands in a pod
kubectl exec -it <pod-name> -- bash

# Delete resources
kubectl delete pod <pod-name>

Getting Started with Kubernetes

To start learning Kubernetes:

  1. Install Docker (if not already installed)
  2. Install kubectl (Kubernetes command-line tool)
  3. Set up a local Kubernetes cluster using Minikube or Docker Desktop
  4. Deploy your first application using kubectl
